## English Version 

### Overview 

The File Archive Kit Plugin ("Plugin") is designed to facilitate local file compression and decompression operations for various archive formats including .zip, .7z, and tar.gz, while prioritizing user privacy and data security. This privacy policy explains how we handle your data when you use our plugin within the Dify platform. 

### License Information 

This plugin is licensed under Apache License 2.0, which is a permissive open-source license that allows for the use, modification, and distribution of software. The full license text is available in the LICENSE file included with this plugin. 

### Data Collection 

**What we DO NOT collect:** 
- Personal identification information 
- User account details 
- Usage analytics or tracking data 
- Device information 
- Location data 
- Cookies or similar tracking technologies 

**What we process:** 
- Files you choose to compress or decompress (.zip, .7z, tar.gz files)
- File metadata (e.g., filename, size, format) required for compression/decompression operations 
- Passwords for encrypted archives (for compression and decompression)
- Temporary processing data required for file operations 

### Data Processing 

**Local Processing:** 
- All compression and decompression operations are performed locally on your system without external data transmission 
- No files are uploaded to external servers during the compression/decompression process 
- Temporary files are created and deleted securely during processing 
- No API keys or external services are required for any operations 

### Data Storage 

**No Data Retention:** 
- We do not store or retain copies of your files after compression or decompression 
- We do not access or view the contents of your files beyond what is necessary for compression/decompression 
- Temporary files are automatically deleted after the compression/decompression process completes 
- No data is transmitted to any third-party services 
- Passwords for encryption are only used during the compression process and are not stored 

### Security Measures 

**Data Protection:** 
- All file processing is performed in a secure local environment 
- No network connections are made for compression/decompression operations 
- Temporary files are handled with appropriate security measures 
- No external API calls or service dependencies 
- AES-256 encryption is used for password-protected archives 
- Strong random password generation using Python's secrets module for auto-generated passwords

## 中文版本 

### 概述 

文件压缩与解压插件（"插件"）旨在促进本地文件压缩和解压操作，支持.zip、.7z和tar.gz等多种归档格式，同时优先保护用户隐私和数据安全。本隐私政策说明了您在Dify平台内使用我们的插件时，我们如何处理您的数据。 

### 许可证信息 

本插件采用Apache License 2.0许可证，这是一个宽松的开源许可证，允许使用、修改和分发软件。完整的许可证文本可在本插件附带的LICENSE文件中找到。 

### 数据收集 

**我们不收集的内容：** 
- 个人身份信息 
- 用户账户详细信息 
- 使用分析或跟踪数据 
- 设备信息 
- 位置数据 
- Cookie或类似跟踪技术 

**我们处理的内容：** 
- 您选择压缩或解压的文件（.zip、.7z、tar.gz文件）
- 压缩/解压操作所需的文件元数据（如文件名、大小、格式）
- 加密归档的密码（用于压缩和解压）
- 文件操作所需的临时处理数据 

### 数据处理 

**本地处理：** 
- 所有压缩和解压操作都在您的系统上本地执行，无需外部数据传输 
- 压缩/解压过程中不会将文件上传到外部服务器 
- 处理过程中会安全地创建和删除临时文件 
- 任何操作都不需要API密钥或外部服务 

### 数据存储 

**不保留数据：** 
- 压缩或解压后，我们不存储或保留您文件的副本 
- 除了压缩/解压所需的内容外，我们不访问或查看您文件的内容 
- 压缩/解压过程完成后，临时文件会自动删除 
- 不会将任何数据传输到第三方服务 
- 加密密码仅在压缩过程中使用，不会被存储 

### 安全措施 

**数据保护：** 
- 所有文件处理都在安全的本地环境中执行 
- 压缩/解压操作不会建立网络连接 
- 临时文件通过适当的安全措施进行处理 
- 无外部API调用或服务依赖 
- 使用AES-256加密进行密码保护的归档 
- 使用Python的secrets模块生成强随机密码用于自动生成的密码
